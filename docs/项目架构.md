# AgentProject — 能源行业AI Agent 完整架构总结

---

## 一、项目概览

**项目名称：** AgentProject — Energy Industry AI Agent  
**课程背景：** Varian / University of Melbourne AI Agent Engineering Course  
**代码仓库：** VarianWrynn/AgentProject (branch: wk4)  
**技术栈：** Python 3.14 · FastAPI · LangGraph · Milvus · Redis · OpenAI-compatible LLM · React/Vite 前端

系统是一个面向**能源行业**的多智能体研究报告生成平台，支持两种工作模式：
1. **对话模式（/chat）**：5节点ReAct循环，处理即时问答
2. **深度研究模式（/research/report）**：7节点多智能体流水线，生成完整行业研究报告

## 三、LangGraph 双图架构

### Graph 1 — 旧图（/chat 端点，ReAct循环）

```
RouterNode → PlannerNode → ExecutorNode → ReflectorNode → CriticNode → END
```

| 节点 | 职责 |
|------|------|
| RouterNode | LLM意图分类：general / data_query / market_analysis / research |
| PlannerNode | 生成JSON多步执行计划 |
| ExecutorNode | 按计划调用rag_search / web_search / text2sql |
| ReflectorNode | 评估执行结果，决定continue / replan / done |
| CriticNode | 最终质量审核 |

**Memory：** Redis `react:{sid}:question/plan/steps`，TTL=3600s

### Graph 2 — 新图（/research/report，多智能体）

```
RouterNode
    ↓
ChiefArchitect          ← KEY_1 (LLM规划)
    ↓
DeepScout               ← KEY_2 (asyncio.gather并行搜索)
    ↓
DataAnalyst             ← KEY_3 (Text2SQL + matplotlib)
    ↓
LeadWriter              ← KEY_4 (ThreadPoolExecutor并行写章节)
    ↓
CriticMaster            ← KEY_3 (对抗审核)
    ↓ (quality_score < 0.6 且 iteration < 3 → 回到DeepScout)
Synthesizer             ← KEY_1 (最终组装)
    ↓
END
```

**RE_RESEARCHING防护（三层）：**
1. threshold 0.6（从原始0.75降低，避免永不收敛）
2. iteration ≥ 2 时强制完成
3. 硬上限3次循环（langgraph_agent.py路由函数）

## 四、6个专项Agent详解

### 4.1 ChiefArchitect（研究规划）
- **输入：** question + intent
- **输出：** hypotheses(3个可验证假设), outline(6章节), research_questions(5-8个子问题)
- **demo_mode：** 跳过LLM调用，返回固定1节大纲 + 1个研究问题（节省~15s）
- **fallback：** LLM返回空时使用`_default_outline()`预设6章结构

### 4.2 DeepScout（并行搜索）
- **输入：** research_questions + pending_queries（RE_RESEARCHING时）
- **输出：** raw_sources, facts（去重后）
- **核心机制：** `asyncio.new_event_loop()` + `asyncio.gather()` 并行执行所有子问题搜索
- **搜索渠道：** Bocha中文网搜 + Milvus RAG向量检索（通过MCP调用，有fallback直连）
- **去重：** SHA-256对(title+snippet)内容去重
- **demo_mode：** 限制1个问题，跳过`_extract_facts()` LLM调用

### 4.3 DataAnalyst（数据分析+图表）
- **输入：** outline + intent
- **输出：** data_points[], charts_data[]（含base64 PNG）
- **SQL流程：** 4条Text2SQL查询 → 结果转data_points → matplotlib生成图表
- **图表类型：** bar（≤8行）/ line（>8行）/ pie，通过SQL特征自动判断
- **中文字体：** `_setup_chinese_font()` via `fm.fontManager.ttflist`，优先Microsoft YaHei
- **demo_mode：** 跳过SQL，调用`_generate_demo_chart()`生成预设储能市场装机图（0.5s）

### 4.4 LeadWriter（报告撰写）
- **输入：** outline + facts + data_points + raw_sources
- **输出：** draft_sections{sec_id: content}, references[]
- **并行化：** `ThreadPoolExecutor(max_workers=6)`，所有章节同时触发LLM，完成时间=max(单章时间)
- **单章格式：** 500-800字，概述→详细分析→小结，引用`[来源N]`格式
- **retry机制：** 每章max_retries=2，2s间隔
- **demo_mode：** 写前2章，跳过摘要LLM生成（自动生成短文本）

### 4.5 CriticMaster（对抗审核）
- **输入：** draft_sections + facts
- **输出：** critic_issues[], quality_score(0~1), pending_queries[], phase
- **6类问题检测：** hallucination / missing_source / logic_error / outdated / incomplete / bias
- **触发RE_RESEARCHING：** quality_score < 0.6 + pending_queries非空
- **demo_mode：** 顶部早返回，quality_score=0.75，phase=done（零LLM调用）

### 4.6 Synthesizer（最终整合）
- **输入：** draft_sections + critic_issues + outline
- **输出：** final_answer（完整Markdown报告）
- **修订流程：** 针对high/medium问题调用`_apply_revisions()`定向修改
- **demo_mode：** 跳过LLM修订，直接组装Markdown

## 五、共享状态（AgentState TypedDict）

```python
class AgentState(TypedDict):
    # 核心
    session_id:         str
    question:           str
    intent:             str
    final_answer:       str
    
    # 规划层
    hypotheses:         list
    outline:            list          # [{id, title, description, keywords}]
    research_questions: list
    
    # 知识层
    facts:              list          # [{content, source, credibility}]
    raw_sources:        list          # [{title, url, snippet, date, source_type}]
    data_points:        list          # [{metric, value, query, sql}]
    
    # 输出层
    draft_sections:     dict          # {sec_id: content_str, "summary": str}
    charts_data:        list          # [{title, type, data, image_b64, query}]
    references:         list          # [{title, url, date}]
    
    # 审核层
    critic_issues:      list          # [{type, description, severity, section_id}]
    pending_queries:    list
    quality_score:      float
    
    # 流控层
    phase:              str           # planning/researching/analyzing/writing/reviewing/done
    iteration:          int           # RE_RESEARCHING循环计数
    demo_mode:          bool          # True=快速演示模式
    
    # ReAct专用
    plan:               dict
    steps_executed:     list
    reflection:         str
    confidence:         float
```

**关键设计：** `demo_mode: bool` 必须在TypedDict中声明，否则LangGraph节点间的partial state update会静默丢弃该字段。

## 六、LLM路由层（llm_router.py）

```python
# Key分配策略
KEY_1 → router, chief_architect, synthesizer   # 低并发，规划型
KEY_2 → deep_scout                             # 高并发，parallel搜索
KEY_3 → data_analyst, critic_master            # 中等并发
KEY_4 → lead_writer                            # 高并发，parallel写作

# 降级链（永不崩溃）
KEY_N → KEY_1 → OPENAI_API_KEY

# 接口
make_llm(role) -> LLMClient    # 用于langgraph_agent.py节点
get_client(role) -> (OpenAI, model)  # 用于直接OpenAI调用
```

LLMClient（react_engine.py）已扩展支持`api_key`/`model`/`base_url`可选构造参数，与旧`LLMClient()`调用完全兼容。

## 七、RAG Pipeline

- **向量库：** Milvus（IVF_FLAT/COSINE），Collection: `knowledge_base`
- **嵌入模型：** BAAI/bge-m3（1024维），CPU推理
- **分块策略：** ParagraphChunker，512 token，50 token overlap，SHA-256去重
- **检索配置：** top_k=5，score threshold=0.45
- **数据源：** 6份能源行业文档（光伏/储能/风电/碳市场等）

---

## 八、Text2SQL Pipeline

- **数据库：** SQLite `data/energy.db`（sales + products表）
- **流程：** 歧义检测 → SQL生成 → 结果摘要
- **term_dict：** 中文业务术语→SQL映射（如"营收"→`SUM(amount)`）
- **Schema：** `sales(id, product, region, amount, sale_date)` + `products(id, name, category, unit_price)`

## 九、SSE实时推送机制

```python
# api_server.py
@app.get("/research/stream")
async def research_stream(session_id: str):
    async def event_generator():
        while True:
            event = redis.blpop(f"sse_events:{session_id}", timeout=600)
            yield f"data: {json.dumps(event)}\n\n"
    return StreamingResponse(event_generator(), media_type="text/event-stream")

# langgraph_agent.py 各节点
def _push_sse_event(sid, type, message, step):
    redis.rpush(f"sse_events:{sid}", json.dumps({...}))
```

前端通过`EventSource`订阅，节点进度实时展示（thinking→searching→analyzing→writing→reviewing→done）。
